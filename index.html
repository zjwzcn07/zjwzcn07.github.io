NTRU涉猎笔记

Lattice

定义：

已知vl，v2，⋯，vm∈R，是一组线性无关向量，
由vl，v2，⋯，vm的整系数线性组合生成的集合为 L={v∈R|v=a1v1+a2v2+⋯+amvm;al,a2,...,am∈z};

性质：

离散性

S为Rn的子集合，若S中不含聚点则称S为离散的．
即对任意X∈S存在r>0,使得以x为球心,r为半径的球B(x，r)={y∈Rn：||x—y||<r} 与S交集只有x.

命题1

L(R的真子集)是一个格当且仅当L是Rn的离散加法子群。

命题2

若vl,v2,⋯,vm∈R”生成的加法群是离散的，
则要么vl,v2,⋯,vm∈ Q，要么vl,v2,⋯,vm ∈Rn是线性无关的．

命题3

格L的任意两组基的过渡矩阵为整数矩阵，且其行列式值为土1。

命题4

令L(R的真子集)为一个秩为m的格，v1,v2,⋯,vm为L的一组基．格L关于这组基的基本域为
F(vl,v2,⋯,vm)={tlvl+t2v2+···+tmvm：0≤ti<1}.

困难问题

SVP-格中最短向量问题

给定秩为d的格L(L是Zn的子集)中的一组基v1,...,vd,寻找一个非零向量u∈L使其满足||u|| = min||v||， v∈L\{0}
这是困难的！NP-hard

CVP-格中最近向量问题

给定秩为d的格L(L是Zn的子集)中的一组基v1,...,vd,目标向量x∈span(L),
寻找一个非零向量u∈L使其满足||x-u|| = min||v||， v∈L
这是困难的！NP-hard

ApprSVP-格中渐进最短向量问题

给定秩为d的格L(L是Zn的子集)中的一组基v1,...,vd,一个渐进因子γ>=1,
寻找一个非零向量u∈L使其满足||u|| <= γmin||v||， v∈L\{0}
这是可以求出足够短但不是最短的向量，复杂度由γ决定

CVP-格中最近向量问题

给定秩为d的格L(L是Zn的子集)中的一组基v1,...,vd,一个渐进因子γ>=1,目标向量x∈span(L),
寻找一个非零向量u∈L使其满足||x-u|| = γmin||v||， v∈L
这是可以求出足够近但不是最近的向量，复杂度由γ决定

cryptopals_site5_Diffie-Hellman

Introduction

Diffie-Hellman是公钥协议中一种安全的密钥交换体制，其安全性基于离散对数的分解困难
以整数环为例
Alice, bob 协商出大素数 p 和 p 的原根 g 
Alice随机选择a, 令 A = (g**a)%p
Bob  随机选择b, 令 B = (g**b)%p
AB可互相发送给对方
于是ALice可计算出 S = (B)**a%p
    Bob  可计算出 S = (A)**b%p
至此二人共享密钥S，而Eavesdroppers由于不能知道ab, 且难以求AB的对数，故不可得S

Implement

Secure Remote Password (SRP)

C&S

Agree on N=[NIST Prime], g=2, k=3, I (email), P (password)

S

random salt, 
xH=SHA256(salt|password) -> Integer x (discard)
v = g**x %N

C->S

Send I, A=g**a % N

S->C

Send salt, B=kv + g**b % N

S, C

uH = SHA256(A|B), u = integer of uH

C

xH=SHA256(salt|password) -> Integer x (discard)
S = (B - k * g**x)**(a + u * x) % N
K = SHA256(S)

S

S = (A * v**u) ** b % N
K = SHA256(S)

C->S

Send HMAC-SHA256(K, salt)

S->C

Send "OK" if HMAC-SHA256(K, salt) validates

Attack

MITM attack

然而这显然是难以抵抗中间人攻击
假设Eve也弄出来两个大素数pa, pb, 
Eve冒充Bob和Alice通信
然后得到明文后再冒充Alice和Bob通信
数学证明从上

break with malicious “g” parameters

when g = 1
KA = 1^a mod p = 1
KB = 1^b mod p = 1
Key = 1^a^b mod p = 1
You can use key = 1 to attack

when g = p
KA = p^a mod p = 0
KB = p^b mod p = 0
Key = p^a^b mod p = 0
You can use key = 0 to attack

when g = p - 1
KA = (p-1)^a mod p = 1 || -1
KB = (p-1)^b mod p = 1 || -1
Key = (p-1)^a^b mod p  = 1 || -1
KA and KB maybe different
You can use key = 1 || p -1  to attack

Break SRP with a zero key

you can send 0 as "A" or multiple of "N" value when you are client

send A = 0
you can know S = (A * v**u) ** b % N
so you can send your S = 0 to attack

send A = multiple of N
you can know S = (k*N * v**u)**b %N = 0
the same as A = 0

Success to attack!
